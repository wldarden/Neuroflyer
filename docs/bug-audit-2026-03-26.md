# NeuroFlyer Bug Audit — 2026-03-26

Comprehensive audit across 4 subsystems: UIManager/screens, evolution/genome, game engine, hangar/components.

---

## Critical (Must Fix)

### 1. Escape double-dispatch: modal AND screen both fire
**Location:** `ui_manager.cpp:82-89`, all modal and screen Escape handlers
**Problem:** `ImGui::IsKeyPressed(ImGuiKey_Escape)` returns true for ALL callers in the same frame. When a modal is open and the user presses Escape, the modal's `on_draw()` catches it AND the screen's `on_draw()` catches it. Result: the modal closes AND the screen pops in one keypress.
**Fix:** `UIManager::draw()` should skip screen's `on_draw()` when a blocking modal is active (use the existing `blocks_input()` API which is currently never checked). Or: screens should guard Escape with `!ui.has_modal()`.

### 2. Double `pop_modal()` in node/layer editor modals
**Location:** `node_editor_modal.cpp:33+191`, `layer_editor_modal.cpp:28+144`
**Problem:** If the user clicks Cancel, `pop_modal()` fires from `draw_ok_cancel()`. Then execution continues to the end of `on_draw()` where the Escape/close check can fire `pop_modal()` again. This pops the wrong modal or crashes on empty stack.
**Fix:** `draw_ok_cancel()` should set a flag instead of calling `pop_modal()` directly. The check at the end of `on_draw()` should be the sole pop site.

### 3. Screens that set `state.current` AND call `push_screen()` create duplicates
**Location:** `hangar_screen.cpp:323-332`, `create_genome_screen.cpp:337-338`, `lineage_tree_screen.cpp:159-200`
**Problem:** Setting `state.current` triggers `sync_legacy_navigation` on the next frame, which pushes ANOTHER screen on top. Result: duplicate screens stacked.
**Fix:** Remove the `state.current = ...` line from any place that also calls `ui.push_screen()`.

### 4. `add_layer`/`remove_layer` corrupts genome-topology gene mapping
**Location:** `evolution.cpp:331-355`
**Problem:** When a layer is inserted/removed, topology indices shift but gene tags (`layer_0_weights`, etc.) keep their old numbering. `rebuild_weight_genes` then writes data to the wrong layer genes, destroying learned weights.
**Fix:** After insert/remove, rename all gene tags to match new topology ordering before calling `rebuild_weight_genes`.

### 5. `evolve_population` crashes on empty population
**Location:** `evolution.cpp:404`
**Problem:** `population.size() - 1` wraps to `SIZE_MAX` (unsigned underflow). Creates distribution `[0, SIZE_MAX]`, causes out-of-bounds access.
**Fix:** Early return if `population.empty()`.

## Important (Should Fix)

### 6. FlySessionScreen Escape uses `go_to_screen()` instead of `pop_screen()`
**Location:** `fly_session_screen.cpp:358-381`
**Problem:** FlySessionScreen remains orphaned on the UIManager stack. Combined with bug #3, creates zombie screens and memory leaks.
**Fix:** Replace `go_to_screen()` with `ui.pop_screen()`.

### 7. `blocks_input()` is declared but never checked
**Location:** `ui_modal.h:17`, `ui_manager.cpp:82-89`
**Problem:** Modals declare `blocks_input() == true` by default, but UIManager never checks it. The screen always gets `on_draw()` even when a blocking modal is open.
**Fix:** Implement the check in `UIManager::draw()`.

### 8. Modals not cleared when parent screen is popped
**Location:** `ui_manager.cpp:27-32`
**Problem:** If a modal was pushed by a screen and that screen gets popped, the modal remains on the stack with dangling references to the destroyed screen's state.
**Fix:** Clear modal stack in `pop_screen()`.

### 9. Stale genes after `remove_layer`
**Location:** `evolution.cpp:295-298`
**Problem:** Genes for removed layers remain in the genome, polluting `flatten()` output and making snapshot weight vectors longer than expected. Also wastes mutation cycles.
**Fix:** `rebuild_weight_genes` must remove genes for layer indices that no longer exist.

### 10. `best_as_snapshot` crashes on empty population
**Location:** `evolution.cpp:495-506`
**Problem:** `max_element` returns `end()` on empty range, dereference is UB.
**Fix:** Guard with empty check.

### 11. `individual_hash` only uses first 8 floats
**Location:** `fly_session_screen.cpp:34`
**Problem:** Related individuals (e.g., after crossover sharing layer_0 weights) get the same hash. MRCA tracker treats them as identical, missing lineage transitions.
**Fix:** Hash all floats or use a strided sample across the full genome.

### 12. `show_delete_confirm_` and pending flags survive screen push/pop
**Location:** `hangar_screen.cpp:33-35`, `variant_viewer_screen.cpp:45-48`
**Problem:** If a user opens a delete dialog, navigates away, and returns, the stale flag causes the dialog to reappear unexpectedly.
**Fix:** Reset `show_delete_confirm_`, `promote_pending_`, `delete_pending_` in `on_enter()`.

### 13. No UI feedback for invalid/duplicate genome names
**Location:** `create_genome_screen.cpp:272-343`
**Problem:** Invalid names silently fail. Duplicate names throw an exception that is caught but not shown to the user.
**Fix:** Show an error message in the UI.

### 14. Names with spaces pass validation
**Location:** `name_validation.h:12-14`
**Problem:** Names like `" "` or `"  foo  "` create directories with invisible space characters.
**Fix:** Trim whitespace and reject all-space names.

### 15. `write_string` truncates at 65535 bytes
**Location:** `snapshot_io.cpp:59`
**Problem:** `static_cast<uint16_t>(s.size())` silently wraps. Corrupt snapshot on load.
**Fix:** Bounds check or use `uint32_t`.

### 16. `count_weights` doesn't include activation values
**Location:** `evolution.cpp:9`
**Problem:** Name is misleading. If used to validate `snap.weights.size()`, it would report too few.
**Fix:** Rename to `count_weights_and_biases` or update to include activations.

---

### 17. Generation ends while last ship is still alive (`alive <= 1`)
**Location:** `fly_session.cpp:498,538`
**Problem:** Evolution triggers when 1 ship remains — that survivor never gets its final score. For `population_size = 1`, the generation ends on the first tick with no gameplay.
**Fix:** End generation when `alive == 0`, not `alive <= 1`.

### 18. Headless phase freezes the window (no SDL event polling)
**Location:** `fly_session.cpp:528-542`
**Problem:** The headless generation loop runs without polling SDL events. The app becomes unresponsive for seconds. User can't press Escape to abort.
**Fix:** Poll events periodically inside the headless loop.

### 19. Occulus renderer alpha overflow
**Location:** `renderers/occulus.cpp:123`
**Problem:** `alpha * 3` overflows `uint8_t` for values > 85. `120 * 3 = 360` wraps to 104 instead of clamping to 255.
**Fix:** `std::min(alpha * 3, 255)`.

### 20. Input vector size can mismatch network after ShipDesign change
**Location:** `sensor_engine.cpp:199-238`, `fly_session.cpp:199`
**Problem:** If ShipDesign changes via pause config but the population topology doesn't, `build_ship_input` produces a different-sized vector than the network expects. UB in `network.forward()`.
**Fix:** Verify `compute_input_size(ship_design) == input_size` after reset.

**Location:** `ship_design.h:64`
**Fix:** Use `unsigned int` and cast.

### 22. `const_cast` in modal editors is unnecessary
**Location:** `node_editor_modal.cpp:111,143`, `layer_editor_modal.cpp:119`
**Fix:** Use non-const `gene()` overload.

**Location:** `ui_manager.cpp:34-42`

### 24. `structural_history` grows unbounded during long training runs
**Impact:** Each histogram uses a `std::map` with ~20 entries of 64-80 bytes each. At 10K gens = ~12-16MB. Not catastrophic but worth capping.

### 25. `promote_to_genome` is not atomic across multiple file writes
**Location:** `genome_manager.cpp:337-378`

### 26. `delete_genome` drops lineage for 2nd+ child genomes
**Location:** `genome_manager.cpp:428-438`

### 27. Towers can spawn at screen edges (x=0 or x=screen_w), clipping off-screen
**Location:** `game.cpp:96`
